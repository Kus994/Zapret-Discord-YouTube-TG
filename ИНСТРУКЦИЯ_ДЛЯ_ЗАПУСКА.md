## 2. Установка Python

### Шаг 2.1: Скачивание

1. Перейдите на [python.org/downloads](https://www.python.org/downloads/)
2. Скачайте **Python 3.12** или **3.13** (рекомендуется 64-bit)
3. Запустите установщик

### Шаг 2.2: Установка (ВАЖНО!)

При установке **обязательно** поставьте галочку:

```
[x] Add Python to PATH
```

Затем нажмите **"Install Now"** (стандартная установка).

### Шаг 2.3: Проверка

Откройте **cmd** (Win + R → `cmd` → Enter) и выполните:

```cmd
python --version
```

Должно вывести:
```
Python 3.12.x
```

Если появляется ошибка "python不是内部或外部命令", значит Python не добавлен в PATH:
- Переустановите Python с галочкой "Add Python to PATH"
- Или добавьте путь вручную: `C:\Users\ВАШЕ_ИМЯ\AppData\Local\Programs\Python\Python312\`

## 4. Запуск из исходников

### Важно: Запускайте из папки проекта!

**НЕ запускайте `run.bat` из `C:\Windows\System32\`** — это вызовет ошибку "Python не найден" и "main.py not found".

### Способ 1: Через run.bat (рекомендуется)

1. Откройте папку проекта: `C:\Users\ВашеИмя\Desktop\KusXXX`
2. Дважды кликните на `run.bat`
3. Скрипт автоматически:
   - Найдёт Python
   - Установит зависимости (если нет)
   - Запустит приложение

### Способ 2: Через cmd

```cmd
:: ВАЖНО: сначала перейдите в папку проекта!
cd C:\Users\ВашеИмя\Desktop\KusXXX
python main.py
```

### Способ 3: С правами администратора

Некоторые функции (оптимизация, сервисы) требуют повышения прав:
1. Нажмите ПКМ на `run.bat`
2. Выберите "Запуск от имени администратора"


### Шаг 5.2: Запустите сборку

```cmd

```

```
Двойной клик на build.bat
```

После сборки файл будет в:


Скопируйте `KUS_Pro.exe` на целевой ПК. Python и зависимости **не требуются**.

## 7. Настройка Telegram Proxy

### Что это?

MTProto-прокси для Telegram через WebSocket (TgWsProxy). Позволяет использовать Telegram даже при блокировках.

### Шаг 7.1: Запустите KUS Pro

Запустите приложение любым способом из раздела 4.

### Шаг 7.2: Перейдите в "Telegram Proxy"

В левом меню нажмите **"Telegram Proxy"**.

### Шаг 7.3: Скачайте TgWsProxy

1. Нажмите **"Скачать TgWsProxy (.exe)"**
2. Подождите завершения загрузки
3. Файл сохранится в папку `tg_proxy/`

### Шаг 7.4: Запустите прокси

Нажмите кнопку **"Запустить"**.

### Шаг 7.5: Настройте Telegram

1. Откройте **Telegram** на телефоне/ПК
2. Перейдите: **Настройки** → **Данные и память** → **Прокси**
3. Нажмите **"Добавить прокси"**
4. Выберите **"MTProto-прокси"**
5. Введите данные:
   - **Сервер:** `127.0.0.1`
   - **Порт:** (см. автоподбор порта ниже)
   - **Secret:** (получите из трей-меню TgWsProxy)

### Шаг 7.6: Получение Secret

1. Найдите иконку TgWsProxy в системном трее (рядом с часами)
2. Нажмите ПКМ на иконку
3. Выберите **"Скопировать ссылку"**
4. Secret будет скопирован в буфер обмена

## 8. Автоподбор порта

### Как это работает

KUS Pro **автоматически** находит свободный порт для Telegram Proxy:

```
Приоритет подбора:
1. Порт из config.json (по умолчанию: 8080)
2. Типичные MTProto порты: 8080, 1443, 443, 8443, 1080, 3128, 8118, 9050
3. Свободный порт в диапазоне 10000-10100
```

### Автоматический режим

При запуске прокси система автоматически:
1. Проверяет настройенный порт
2. Если занят — переключается на следующий свободный
3. Выводит сообщение в лог

### Ручная настройка порта

Откройте `config.json` и измените:

```json
{
  "tg_proxy_port": 8080
}
```

Допустимые порты: 1024-65535

### Проверка порта

В приложении нажмите **"Тест порта"** — проверит, открыт ли указанный порт.

## 9. Конфигурация

### Файл config.json

Все настройки хранятся в `config.json`:

```json
{
  "game_mode": {
    "apps": [
      {
        "label": "Wallpaper Engine",
        "process": "wallpaper64.exe",
        "path": "",
        "enabled": true
      }
    ],
    "disable_transparency": true,
    "enable_taskbar_autohide": true,
    "auto_return_enabled": false,
    "auto_return_minutes": 60,
    "hotkey": ""
  },
  "autostart_zapret": false,
  "autostart_tg_proxy": false,
  "tg_proxy_port": 8080,
  "start_minimized": false,
  "auto_update_enabled": true,
  "auto_update_zapret": true,
  "auto_update_tg_proxy": true,
  "tg_bot_token": "",
  "tg_bot_chat_id": ""
}
```

### Описание параметров

| Параметр | Тип | Описание |
|----------|-----|----------|
| `tg_proxy_port` | int | Порт Telegram Proxy (1024-65535) |
| `autostart_zapret` | bool | Автозапуск Zapret при старте |
| `autostart_tg_proxy` | bool | Автозапуск Telegram Proxy |
| `start_minimized` | bool | Запуск свёрнутым в трей |
| `auto_update_enabled` | bool | Автопроверка обновлений |
| `auto_update_zapret` | bool | Автообновление Zapret |
| `auto_update_tg_proxy` | bool | Автообновление TgWsProxy |
| `game_mode.apps` | array | Список программ для отключения в игровом режиме |
| `game_mode.hotkey` | string | Горячая клавиша для игрового режима |

## 10. Устранение неполадок

### Проблема: "Python не найден"

**Решение:**
1. Переустановите Python с галочкой "Add Python to PATH"
2. Перезапустите cmd
3. Проверьте: `python --version`

### Проблема: "Could not find Qt platform plugin 'windows'"

**Решение:**
```cmd
pip install --upgrade PyQt5
```

### Проблема: PyQt5 не устанавливается (кириллица в пути)

**Решение:**
1. Установите Python в путь без кириллицы: `C:\Python312\`
2. Или создайте junction:
```cmd
mklink /J C:\PyEnv "C:\Users\Кусь\AppData\Local\Programs\Python\Python312\Lib\site-packages\PyQt5"
```

### Проблема: "No matching distribution found for PyQt5"

**Решение:**
```cmd
pip install PyQt5 psutil certifi
```

### Проблема: pip не может скачать пакеты (ProxyError / ConnectionResetError)

**Причина:** Прокси-сервер блокирует подключение к PyPI.

**Как исправить (пошагово):**

1. Откройте **cmd** (Win + R → `cmd` → Enter)
2. Выполните команды по очереди:
```cmd
set HTTP_PROXY=
set HTTPS_PROXY=
cd C:\Users\ВашеИмя\Desktop\KusXXX
pip install PyQt5 psutil certifi
```
3. После успешной установки запустите `run.bat`

**Важно:**
- Не запускайте pip из `C:\Windows\System32\`
- Всегда переходите в папку проекта: `cd C:\Users\ВашеИмя\Desktop\KusXXX`

### Проблема: Порт занят

**Решение:**
Приложение автоматически выберет свободный порт. Проверьте лог:

```
Порт 8080 занят, выбран свободный: 8443
```

Или вручную освободите порт:
```cmd
netstat -ano | findstr :8080
taskkill /PID <номер_процесса> /F
```

### Проблема: TgWsProxy не запускается

**Решение:**
1. Убедитесь, что скачали последнюю версию
2. Проверьте, нет ли блокировки антивирусом
3. Запустите от имени администратора

### Проблема: Telegram не подключается к прокси

**Решение:**
1. Убедитесь, что TgWsProxy запущен (зелёный индикатор)
2. Проверьте порт: нажмите "Тест порта"
3. Используйте "Проверить здоровье"
4. Нажмите "Авто-фикс"

### Проблема: Ошибка "Access Denied" при оптимизации

**Решение:**
Запустите приложение от имени администратора:
- ПКМ на `run.bat` → "Запуск от имени администратора"

### Проблема: run.bat не находит main.py

**Решение:**
1. Убедитесь, что `run.bat` находится в той же папке, что и `main.py`
2. Запускайте `run.bat` из папки проекта, а не из другого места
3. Или используйте прямой запуск через cmd:
```cmd
cd C:\Users\Кусь\Desktop\KusXXX
python main.py
```

### Проблема: Автозапуск Zapret не работает

**Решение:**
1. Включите "Запускать Zapret при старте приложения" в Настройках
2. Запустите Zapret хотя бы один раз с нужным пресетом — он сохранится
3. При следующем запуске KUS Pro Zapret запустится автоматически

### Проблема: Автозапуск Telegram Proxy не работает

**Решение:**
1. Включите "Запускать Telegram Proxy при старте" в Настройках
2. Убедитесь, что TgWsProxy скачан (вкладка "Telegram Proxy")
3. При следующем запуске KUS Pro прокси запустится автоматически
