# Найдена настоящая причина: окно не появлялось из-за UAC

## Диагностика по run_log.txt

Лог показал:
```
[Qt] Found at: C:\Users\...\PyQt5\Qt5\plugins\platforms
```
...и больше ничего. Код завершения — `0`. Это значило: процесс
завершился штатно, без единой ошибки, но окно так и не появилось.

## Настоящая причина

В `modules/elevation.py` (запрос прав администратора через UAC при
старте) было два связанных бага:

**Баг 1 — безусловный `sys.exit(0)`.** Старый код:
```python
def relaunch_as_admin():
    ctypes.windll.shell32.ShellExecuteW(None, "runas", _windowless_python(), params, None, 0)
    sys.exit(0)
```
`ShellExecuteW` вызывает диалог UAC и возвращает управление **сразу**,
не дожидаясь решения пользователя. Сразу следом стоял `sys.exit(0)` —
то есть исходный процесс завершался **независимо от того**, нажал
пользователь «Да» или «Нет» в диалоге UAC, и независимо от того,
удалось ли реально запустить новый процесс. Если UAC был отклонён
(или просто не успел всплыть на экран до завершения) — повышенный
процесс не стартовал, а исходный уже умер. Результат: ни одного окна,
никакой ошибки, чистый код возврата `0` — ровно то, что было в логе.

**Баг 2 — `SW_HIDE` (последний параметр `0` в `ShellExecuteW`).**
Это флаг "запустить скрытым", что в некоторых случаях могло мешать
новому процессу нормально показать своё окно.

## Исправление

`modules/elevation.py` переписан: `relaunch_as_admin()` теперь
**проверяет реальный код возврата** `ShellExecuteW` (значения ≤32 — 
это код ошибки, например 5 = «доступ запрещён», то есть пользователь
отклонил UAC) и возвращает `(success, message)` вместо того, чтобы
сразу завершать процесс. Флаг показа окна исправлен на
`SW_SHOWNORMAL`.

`main.py` → `request_admin()` обновлён соответственно: если повышение
прав не удалось по любой причине — печатает понятную причину в лог
(`run_log.txt` теперь это покажет) и **продолжает работу в текущем
процессе без прав администратора**, вместо того чтобы пропадать
молча. Если же повышение прав удалось — всё работает как раньше,
текущий процесс корректно уступает место новому.

Права администратора при этом всё ещё запрашиваются при каждом
запуске, как и было задумано — изменилось только то, что отказ от
UAC больше не «убивает» приложение целиком.

## Что сделать сейчас

Замени `main.py` и `modules/elevation.py` на версии из этого архива
и запусти `run.bat`. Теперь должно произойти одно из двух:
- появится диалог UAC → нажми «Да» → откроется окно приложения;
- если диалог UAC не появляется вообще или ты его отклоняешь — теперь
  `run_log.txt` покажет строку вида `[UAC] Не удалось запустить с
  правами администратора: ...` и приложение всё равно откроется без
  повышенных прав, а не пропадёт.

Если после этого фикса окно всё ещё не появляется — пришли мне новый
`run_log.txt`, в нём теперь будет явно видно, что происходит с UAC.
