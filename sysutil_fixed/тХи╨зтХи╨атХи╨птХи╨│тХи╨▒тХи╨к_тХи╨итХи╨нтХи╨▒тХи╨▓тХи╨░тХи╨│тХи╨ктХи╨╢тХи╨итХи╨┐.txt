SYSUTIL — ИНСТРУКЦИЯ ПО ЗАПУСКУ НА ЛЮБОМ КОМПЬЮТЕРЕ
=====================================================

Эта инструкция написана для человека, у которого на компьютере
ещё ничего не установлено. Если что-то уже стоит — просто
пропускайте соответствующий шаг.


ШАГ 0. ЧТО ПОНАДОБИТСЯ
-----------------------
- Windows 10 или 11.
- Права администратора на этом компьютере (для установки Python
  и для части функций самой утилиты).
- Интернет (для скачивания Python и зависимостей).


ШАГ 1. УСТАНОВКА PYTHON
------------------------
1. Откройте браузер и перейдите на https://www.python.org/downloads/
2. Нажмите большую кнопку "Download Python 3.x.x" (любая версия
   3.10 и выше подходит).
3. Запустите скачанный установщик.
4. ВАЖНО: на первом экране установки внизу обязательно поставьте
   галочку "Add python.exe to PATH" (или "Add Python to PATH").
   Если её не поставить — Windows не будет знать, где искать
   команду python, и все следующие шаги не сработают.
5. Нажмите "Install Now" и дождитесь окончания установки.
6. Перезапустите компьютер (не всегда обязательно, но если что-то
   не заработает на шаге 3 — это первое, что нужно сделать).


ШАГ 2. ПРОВЕРКА УСТАНОВКИ PYTHON
---------------------------------
1. Откройте командную строку: нажмите клавишу Win, введите "cmd",
   нажмите Enter.
2. Введите команду:
       python --version
3. Если в ответ появилась строка вида "Python 3.12.4" — всё хорошо,
   переходите к шагу 3.
4. Если появилась ошибка "python не является внутренней или внешней
   командой" — Python установлен без галочки PATH. Решение:
   переустановите Python (шаг 1), не забыв галочку, либо вручную
   добавьте путь к Python в переменные среды (Win → "переменные
   среды" → Path → добавить путь к папке Python).


ШАГ 3. ПОЛУЧЕНИЕ ПАПКИ С ПРОГРАММОЙ
-------------------------------------
1. Скопируйте папку sysutil (целиком, со всеми вложенными папками
   modules и assets) на компьютер, где будете запускать —
   например, на Рабочий стол или в Документы.
2. Откройте эту папку в Проводнике и убедитесь, что внутри лежат:
       main.py
       requirements.txt
       README.md
       modules/   (папка)
       assets/    (папка)


ШАГ 4. УСТАНОВКА ЗАВИСИМОСТЕЙ
-------------------------------
1. Откройте папку sysutil в Проводнике.
2. Кликните один раз на адресную строку Проводника (наверху окна,
   где написан путь) — путь выделится синим.
3. Скопируйте путь (Ctrl+C).
4. Откройте командную строку (Win → "cmd" → Enter).
5. Введите "cd " (cd и пробел, без кавычек), затем вставьте путь
   (правый клик мышью или Ctrl+V), нажмите Enter.
   Командная строка должна показывать путь к папке sysutil.
6. Введите команду:
       pip install -r requirements.txt
7. Подождите, пока скачаются и установятся три библиотеки:
   customtkinter, pillow, psutil. Если все три уже стояли — pip
   напишет "Requirement already satisfied" по каждой, это нормально.


ШАГ 5. ЗАПУСК УТИЛИТЫ
-----------------------
Есть два способа запуска — выберите любой.

СПОСОБ А (просто двойной клик, без окна консоли вообще):
1. В папке sysutil дважды кликните по файлу "Запуск.vbs".
2. Появится системное окно Windows с запросом прав администратора
   (UAC) — нажмите "Да".
3. Откроется только окно программы — никакого чёрного окна
   консоли рядом не будет.

СПОСОБ Б (через командную строку, как раньше):
1. В командной строке (вы уже находитесь в папке sysutil) введите:
       pythonw main.py
   (именно "pythonw", а не "python" — это и есть версия Python без
   консольного окна; если набрать "python main.py", рядом с окном
   программы будет видно чёрное окно консоли).
2. Появится системное окно Windows с запросом прав администратора
   (UAC) — нажмите "Да".
3. Откроется окно программы с фоновым изображением и меню слева.

В обоих случаях при нажатии "Да" в окне UAC программа на мгновение
"перезапускается" с правами администратора — это нормально, окно
программы появится снова через секунду-две.


ЧТО ДЕЛАТЬ, ЕСЛИ ЧТО-ТО НЕ РАБОТАЕТ
-------------------------------------
- Ошибка "No such file or directory: requirements.txt" —
  вы не находитесь в папке sysutil в командной строке.
  Повторите шаг 4, пункты 2-5.
- Ошибка после "python main.py" с текстом Traceback —
  скопируйте полный текст ошибки и пришлите тому, кто помогал
  собирать программу, либо найдите в коде main.py строку,
  указанную в ошибке (там будет "line XXX").
- Окно программы открылось, но кнопки не реагируют на сетевые
  или системные действия — скорее всего, вы отклонили запрос UAC
  на шаге 5.2. Закройте программу и запустите её снова, на этот
  раз нажав "Да" в окне запроса прав.
- Программа не открывается совсем, окно командной строки мигает
  и закрывается — откройте cmd отдельно (как в шаге 2) и запустите
  "python main.py" из него вручную, тогда ошибка останется видна
  на экране и её можно будет прочитать.


ПЕРЕНОС НА ДРУГОЙ КОМПЬЮТЕР
-----------------------------
Просто скопируйте всю папку sysutil на флешку или через облако
и повторите шаги 1, 2, 4, 5 на новом компьютере (Python нужно
ставить на каждом компьютере отдельно — он не "переносится"
вместе с папкой).
